# Professional Backend Implementation Plan - Next Idea Solutions

## Overview

This plan outlines the implementation of a comprehensive backend system for the Next Idea Solutions Next.js application using Next.js API Routes, MySQL database with mysql2 driver, and JWT-based authentication with secure session management.

**Current State:** The application is a static marketing/portfolio site with no internal backend, no database, no authentication, and relies on an external PHP API at `https://demo.nextideasolution.com/api` for portfolio data.

**Goal:** Build a complete backend infrastructure to replace the external PHP API, add authentication, contact form handling, analytics, and a full admin CMS.

## Architecture

### Tech Stack
- **Framework:** Next.js 16.0.10 (existing)
- **Database:** MySQL 8.0+
- **ORM:** mysql2 (raw SQL with prepared statements)
- **Authentication:** JWT with HTTP-only cookies
- **Security:** bcryptjs, helmet, rate limiting, input validation
- **Session Management:** Secure cookies with refresh token rotation

## API Endpoints

### Authentication Endpoints

#### POST /api/auth/register
- **Access:** Public
- **Body:** `{ username, email, password }`
- **Validation:** Username 3-50 chars, valid email, password 8+ chars with complexity
- **Response:** `{ success, message, user: { id, username, email, role } }`
- **Security:** Password hashing with bcrypt (12 rounds), rate limited

#### POST /api/auth/login
- **Access:** Public
- **Body:** `{ email, password }`
- **Response:** `{ success, user: { id, username, email, role } }`
- **Security:** Sets HTTP-only cookie with JWT, rate limited, timing-safe comparison
- **Cookies:** 
  - `access_token` (HTTP-only, Secure, SameSite=Strict, 15min expiry)
  - `refresh_token` (HTTP-only, Secure, SameSite=Strict, 7day expiry)

#### POST /api/auth/logout
- **Access:** Authenticated
- **Security:** Revokes refresh token, clears cookies
- **Response:** `{ success, message }`

#### POST /api/auth/refresh
- **Access:** Public (valid refresh token required)
- **Security:** Refresh token rotation, old token invalidated
- **Response:** New access token cookie

#### POST /api/auth/forgot-password
- **Access:** Public
- **Body:** `{ email }`
- **Security:** Rate limited, no user enumeration
- **Response:** `{ success, message }` (generic message)

### Portfolio Endpoints

#### GET /api/portfolio
- **Access:** Public (read), Admin/Editor (write via POST)
- **Query:** `?page=1&limit=10&category_id=&search=&lang=en&featured=`
- **Response:** `{ success, data: { items, pagination } }`
- **Caching:** ISR with revalidation

#### POST /api/portfolio
- **Access:** Admin, Editor
- **Body:** `{ title, description, category_id, client_name, client_website, project_date, featured, images[], technologies[] }`
- **Validation:** Required fields, image URL validation
- **Response:** `{ success, data: { portfolio_item } }`

#### GET /api/portfolio/[id]
- **Access:** Public
- **Response:** `{ success, data: { item, images, technologies } }`

#### PUT /api/portfolio/[id]
- **Access:** Admin, Editor
- **Body:** Same as POST (partial updates supported)
- **Response:** `{ success, data: { portfolio_item } }`

#### DELETE /api/portfolio/[id]
- **Access:** Admin
- **Response:** `{ success, message }`

#### POST /api/portfolio/[id]/images
- **Access:** Admin, Editor
- **Body:** `{ images: [{ url, alt_text, sort_order, is_primary }] }`
- **Response:** `{ success, data: { images } }`

## Security Implementation

### 1. Authentication Security
```javascript
// JWT Configuration
- Access Token: 15 minutes expiry
- Refresh Token: 7 days expiry
- Algorithm: HS256
- Secret: Environment variable (min 32 chars)
- Cookie: HTTP-only, Secure, SameSite=Strict
```

### 2. Password Security
```javascript
- Hashing: bcrypt with 12 rounds
- Minimum password: 8 characters
- Complexity: 1 uppercase, 1 lowercase, 1 number, 1 special char
- No password reuse (last 3 passwords tracked)
```

### 3. Rate Limiting
```javascript
// Using memory-based rate limiter (Redis optional for production)
- Login: 5 attempts per 15 minutes per IP
- Registration: 3 attempts per hour per IP
- Contact form: 3 submissions per hour per IP
- API general: 100 requests per 15 minutes per IP
- Admin routes: 50 requests per 15 minutes per authenticated user
```

### 4. Input Validation
```javascript
// All inputs validated and sanitized
- SQL injection: Prepared statements (mysql2)
- XSS: HTML entity encoding on output
- CSRF: SameSite cookies + CSRF tokens for state-changing operations
- Email validation: Regex + DNS check
- URL validation: Allowlist for portfolio items
```

### 5. HTTP Security Headers
```javascript
// Via middleware
- X-Content-Type-Options: nosniff
- X-Frame-Options: DENY
- X-XSS-Protection: 1; mode=block
- Strict-Transport-Security: max-age=31536000; includeSubDomains
- Content-Security-Policy: Configured for Next.js
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy: camera=(), microphone=(), geolocation=()
```

### 6. Session Security
```javascript
- Token rotation on refresh
- Concurrent session limits (configurable)
- Session invalidation on password change
- IP binding option for refresh tokens
- User-Agent fingerprinting for anomaly detection
```

### 7. File Upload Security
```javascript
- Allowed types: image/jpeg, image/png, image/webp, image/gif
- Max file size: 5MB
- Filename sanitization
- Magic number verification
- Image dimension validation
- EXIF data stripping
```
