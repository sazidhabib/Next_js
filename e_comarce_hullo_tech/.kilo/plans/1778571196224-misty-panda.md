# Hero Banner Slider/Poster Implementation Plan

## Goal
Replace the existing hero banner carousel in src/app/page.jsx with a new slider/poster design while maintaining all existing functionality and not changing any other components or styles in the project.

## Current Implementation Analysis
- Located in src/app/page.jsx lines 47-112
- Uses banners array (lines 10-32) with objects containing: id, title, subtitle, image, bgColor
- Manual navigation with prev/next buttons and dot indicators
- Each slide contains: title, subtitle, CTA button, and image in a two-column layout
- Responsive design with Tailwind classes
- Uses Next.js Image component for optimization

## Proposed Solution: Poster Slider
Transform each banner slide into a poster-like design where:
- Background image takes full slide height with subtle overlay
- Title, subtitle, and CTA are positioned prominently over the image
- Maintains same navigation controls (prev/next buttons, dot indicators)
- Preserves responsive behavior
- Uses existing color variables and styling from globals.css

## Implementation Steps
1. Modify the banners data structure if needed to support poster design (may add poster-specific fields)
2. Replace the hero banner section (lines 47-112) with new poster slider implementation
3. Each slide will:
   - Use full-bleed background image with dark overlay for text readability
   - Position title/subtitle/CTA in a prominent location (likely left/top or center)
   - Maintain clickable CTA button linking to /offers
   - Preserve background color concept as accent/tint
4. Keep identical navigation controls:
   - Prev/next buttons with same styling
   - Dot indicators with same active/inactive states
5. Ensure responsive behavior matches current implementation
6. Import any additional assets if needed (but reuse existing images where possible)

## Files to Modify
- src/app/page.jsx (only the hero banner section)

## Styling Approach
- Use existing Tailwind utilities from globals.css
- Leverage existing color variables (--color-primary, etc.)
- Maintain same hover/transition effects
- Ensure no changes to global styles or other components

## Validation Criteria
- Visual design resembles promotional posters
- All existing navigation functionality preserved
- Responsive behavior maintained (mobile/desktop)
- No changes to other components (Navbar, CategoryGrid, FeaturedProducts, etc.)
- No changes to global styles in globals.css or Tailwind config
- Existing image assets continue to work
